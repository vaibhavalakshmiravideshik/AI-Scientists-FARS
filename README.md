# AI-Scientists-FARS

## Overview

This repository organizes the paper generation outputs associated with the FARS project. It brings together the proposals written from FARS GitHub repositories, the paper sets produced by several AI scientist systems, the reviewer outputs used for comparison, and the analysis files derived from those evaluations.

At a high level, this is a repository about proposal to paper generation. The FARS proposals are research proposal style documents created from FARS GitHub repositories. Those proposals were then used to produce papers with several different automated systems so that their outputs could be compared side by side.

## What the FARS proposals are

The proposals in this repository are project level research writeups derived from FARS GitHub repositories. They serve as the starting point for downstream paper generation. In practical terms, they are the source material that different paper writing systems were asked to turn into papers.

The repository currently includes 190 proposal markdown files under `FARS/fars_proposals`. Not every proposal appears as a generated paper in every folder. The main full paper collection represented elsewhere in the repository contains 167 proposal linked papers, which is why some counts differ across folders.

One important detail is that only a limited number of FARS GitHub repositories include readily usable datasets. That is especially relevant for `Data-to-Paper`, because this system depends more directly on dataset availability than the other systems do. As a result, the `Data-to-Paper` outputs are much fewer in the larger full proposal collection than the outputs from the other systems.

## Repository structure

### `FARS`

This folder contains the core FARS material.

`FARS/fars_proposals` contains 190 proposal markdown files.

`FARS/fars_papers` contains 167 paper PDFs associated with the proposal set that was carried forward into the larger comparison.

These files are the closest thing to the source corpus for the rest of the repository.

### `AI-Scientist-generated-papers`

This folder contains the papers generated by different AI scientist systems from the FARS proposal set.

It is divided into two collections.

#### `AI-Scientist-generated-papers/pilot-paper-set`

This is the smaller generated paper collection. It contains five system folders with 15 papers each.

`Cycle Researcher` contains 15 papers.

`Data-to-Paper` contains 15 papers.

`FARS` contains 15 papers.

`Sakana v1` contains 15 papers.

`Sakana v2` contains 15 papers.

`Results` contains the evaluation outputs for this 15 paper collection, including summary tables, score files, per paper JSON outputs, and plots comparing the systems.

These results apply only to the pilot paper set. A corresponding results folder for the larger full proposal collection has not been added yet.

#### `AI-Scientist-generated-papers/full-proposal-set`

This is the larger full proposal collection.

`Cycle Researcher_full` contains 167 papers.

`Sakana v1_full` contains 167 papers.

`Sakana v2_full` contains 167 papers.

`Data-to-Paper_full` contains 33 papers.

The smaller `Data-to-Paper_full` count is expected. Many FARS repository links do not provide datasets in a way that supports that pipeline, so only a subset of proposals could be turned into Data-to-Paper outputs.

### `AI-Reviewer-outputs`

This folder stores reviewer side outputs associated with the generated papers and their evaluation workflow.

### Root level `README.md`

This file is meant to help readers quickly understand what the repository contains and why the folder counts are different across systems.

## Why the folder counts differ

The counts are not supposed to match everywhere.

The proposal corpus contains 190 proposal markdown files.

The larger shared paper comparison set contains 167 papers for FARS, Cycle Researcher, Sakana v1, and Sakana v2.

The `Data-to-Paper` system has only 33 papers in the larger collection because dataset access is a practical bottleneck for many of the linked repositories.

The smaller paper collection contains 15 papers per system and was kept as a separate group in this repository.

## Results and analysis files

The analysis files inside `AI-Scientist-generated-papers/pilot-paper-set/Results` include score summaries, proposal score tables, pairwise comparisons, per paper JSON outputs, and figure files. These are the processed outputs used to compare the generated papers across systems rather than raw papers themselves.

At the moment, those result files cover only the 15 paper pilot collection. Results for the larger full proposal collection are still to be added.

## In short

If you are trying to navigate the repository quickly, the easiest way to think about it is this.

`FARS/fars_proposals` is the proposal source set.

`FARS/fars_papers` is the corresponding FARS paper set.

`AI-Scientist-generated-papers` is the generated paper collection from multiple systems.

`AI-Reviewer-outputs` is where the reviewer side material lives.

`Results` folders contain the comparison outputs built from the generated papers.
