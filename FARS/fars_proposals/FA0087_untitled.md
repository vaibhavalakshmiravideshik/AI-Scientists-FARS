# RazorSFT: KL-Minimal On-Policy Target Selection for Continual Post-Training

## Scope and Constraints

- **Paper Type**: Short paper
- **Target Venues**: NeurIPS, ICML, ICLR, ACL, EMNLP (or similar top AI conferences)

## Introduction

### Context and Motivation

Large language models (LLMs) are often updated repeatedly after pretraining (e.g., to add new tools, domains, or policies). In these sequential update settings, a common failure mode is **catastrophic forgetting**: after training on a later stage, the model’s performance on earlier capabilities drops.

This degradation has been documented in LLM continual learning benchmarks such as **TRACE**, which evaluates sequential fine-tuning across diverse tasks and additionally measures changes in general abilities and safety.

Recent 2025–2026 evidence suggests that forgetting in post-training is strongly influenced by the *distribution of training targets*. In particular, multiple works report that **on-policy learning** (learning from samples generated by the current model) can preserve prior abilities substantially better than **offline supervised fine-tuning (SFT)** on a fixed dataset.

### The Problem

A practitioner-friendly default for adaptation is **offline SFT**, which minimizes cross-entropy on fixed prompt–target pairs (human-written or teacher-generated). Offline SFT is operationally simple, but can induce large distribution shift away from the initial model, which appears correlated with forgetting.

A strong alternative is **reinforcement learning with verifiable rewards (RLVR)**, where training uses automated pass/fail checks (e.g., unit tests) as reward. A widely used RLVR algorithm is **GRPO (Group Relative Policy Optimization)**. Recent work shows GRPO can substantially reduce forgetting, but it typically requires additional RL infrastructure (rollout generation loops, advantage normalization, KL penalties for stability), which is not always available in standard SFT pipelines.

This proposal targets a decision-relevant question:

> Can an **SFT-compatible procedure** match GRPO’s forgetting resistance in a sequential post-training curriculum, by choosing training targets that are both **correct** and **close (in KL)** to the current policy?

We include strong non-training baselines (zero-shot prompting and best-of-\(N\) inference) to ensure the effect cannot be matched by inference-time sampling alone.

### Key Insight and Hypothesis

**Key insight (mechanism hypothesis):** For tasks with an automatic verifier, there are typically multiple correct outputs per prompt. **RL’s Razor** argues that forgetting is predicted by the KL divergence from the base model on the new task; on-policy RL tends to bias toward solutions that achieve correctness with smaller KL shift. Motivated by this, we propose to approximate “KL-minimal correct solutions” in an SFT objective by selecting, among verifier-passing candidates, the one with the **highest probability under the current model**.

**Hypothesis:** In a sequential post-training curriculum, **RazorSFT** (verifier-filtered on-policy target selection + reference fallback) will achieve retention comparable to **GRPO** at similar final task performance, and will outperform **offline SFT** on forgetting metrics.

**Why this might fail:** (i) the model may rarely produce verifier-passing outputs, making RazorSFT degenerate to offline targets; (ii) the verifier filter may be unnecessary and on-policy targets alone might explain any gains; (iii) gains might come from implicit training-intensity changes (fewer effective updates) rather than distributional effects.

## Proposed Approach

### Overview

We propose **RazorSFT**, an SFT-compatible target construction procedure for **verifiable** tasks. For each training prompt, we sample multiple **on-policy** candidate responses from the current model, filter candidates by an automatic verifier, and then train with standard cross-entropy on a chosen target. If no candidate passes the verifier, we fall back to a fixed offline reference target to keep training intensity constant.

### Method Details

**Notation (defined for the rest of the proposal):**
- Prompts: \(\{x_i\}\)
- Offline reference targets: \(y_i^{\mathrm{ref}}\) (e.g., teacher outputs)
- Current model policy before epoch \(e\): \(\pi_{\theta_{e-1}}(\cdot\mid x)\)
- Task verifier: \(V_t(x,y)\in\{0,1\}\)

**RazorSFT-\(k\) (default \(k=8\), per epoch):**
1. For each prompt \(x_i\), sample \(k\) candidates \(y_{i,1..k} \sim \pi_{\theta_{e-1}}(\cdot\mid x_i)\).
2. Accepted set \(\mathcal{A}_i = \{y_{i,j}: V_t(x_i,y_{i,j})=1\}\).
3. Choose target:
   - If \(\mathcal{A}_i\neq\emptyset\):
     \[
     y_i^* = \arg\max_{y\in\mathcal{A}_i} \log \pi_{\theta_{e-1}}(y\mid x_i)
     \]
     (the **highest-probability correct** sample; intended to be closer in KL than arbitrary correct samples).
   - Else: \(y_i^*=y_i^{\mathrm{ref}}\) (**fallback**).
4. Run standard SFT on \((x_i,y_i^*)\).

**Diagnostics (recorded, not treated as extra experimental conditions):**
- Replacement rate: \(\rho_t = \mathbb{E}_i[\mathbb{1}(y_i^*\neq y_i^{\mathrm{ref}})]\).
- Mean chosen-target log-probability under \(\pi_{\theta_{e-1}}\) (proxy for target/policy mismatch).

**Interpretability threshold (pre-registered):** If \(\rho_t < 0.5\) for any stage under RazorSFT-8, treat the run as **inconclusive** for the claim “KL-minimal on-policy targets can replace RLVR”, because the method is effectively training mostly on offline targets.

### Key Innovations

1. **KL-minimal correct target selection**: choose the *highest-probability verifier-passing* target, rather than training on arbitrary correct generations.
2. **Reference fallback to fix training intensity**: every prompt contributes exactly one target, reducing the confound “retention improved because we trained on fewer/easier examples.”
3. **Direct comparison to GRPO in a sequential curriculum**: evaluate whether an **SFT-compatible procedure** can match RLVR retention in the setting practitioners care about (repeated updates).

## Related Work

### Field Overview

LLM continual learning methods can be grouped into: (i) **replay/rehearsal** (mixing old data during training), (ii) **regularization/gradient shaping** (constraining updates to preserve prior behavior), (iii) **parameter isolation / adapters** (task-specific parameters such as LoRA), and (iv) **data-distribution approaches**, including on-policy learning and distillation.

For LLMs, forgetting is often evaluated with task matrices (performance on each task after each stage) and summary statistics such as **backward transfer (BWT)** and **forgetting measure (FM)**. Benchmarks like TRACE extend evaluation to general abilities and safety.

This proposal focuses on the **data-distribution** axis: selecting training targets that are correct and close to the current policy, motivated by KL-based analyses of forgetting.

### Closest Prior Work

1. **Retaining by Doing** shows that on-policy data is central and that **Iterative-SFT** (regenerating targets at epoch boundaries) reduces forgetting; RazorSFT differs by selecting the **highest-probability correct** target (explicit KL bias) and using **fallback** to keep per-prompt training intensity fixed.
2. **RL’s Razor** motivates KL-minimal solutions and shows that SFT can match RL when trained on an oracle KL-minimal distribution; RazorSFT proposes an implementable approximation for verifiable tasks by selecting high-probability correct targets from on-policy samples.
3. **RFT naturally mitigates forgetting** demonstrates strong continual retention for GRPO/ReMax/RLOO; we treat GRPO as the strongest baseline and test whether an **SFT-compatible procedure** can match it.
4. **SDFT** achieves on-policy learning from demonstrations via reverse-KL distillation; RazorSFT instead uses verifiers to define correctness and does not rely on in-context teacher behavior.

## Experiments

### Experimental Setup

**Curriculum (3 stages):** We use a minimal sequential curriculum of 3 tasks with automatic evaluation:
1) **IFEval**: instruction-following constraints evaluated by a rule-based checker.
2) **MMLU-style multiple choice**: accuracy via exact-match on selected option.
3) **Countdown**: arithmetic equation construction with a programmatic validity checker.

Each stage starts from the checkpoint produced by the previous stage.

**Base models:**

| Model | Size | Download Link | Notes |
|---|---:|---|---|
| Qwen2.5-Instruct | 7B | https://huggingface.co/Qwen | Use the official 7B instruct checkpoint; keep model fixed across all conditions |

**Baseline Ladder (REQUIRED):**
- **Zero-shot prompting** on each task (no training).
- **Best-of-8 inference** on each task (inference-time scaling) to quantify gains achievable without any weight updates.
- **Offline SFT** (standard baseline for sequential post-training).
- **GRPO (RLVR)** as the strongest known training baseline for verifiable tasks.
- **RazorSFT** (proposed) as an SFT-compatible alternative to GRPO.

**Training conditions (main; 3 conditions):**
1. **Offline SFT** on \(y^{\mathrm{ref}}\) targets.
2. **GRPO** with verifier reward \(r\in\{0,1\}\) (group size \(n=8\)).
3. **RazorSFT-8** (\(k=8\)).

**Training ablation (1 condition; run 1 seed initially):**
4. **OnPolicy-SFT (unfiltered)**: train on a single on-policy sample per prompt without verifier filtering (tests whether filtering is necessary).

**Training data:** We use the prepared splits and teacher targets from the *Retaining by Doing* repository to avoid dataset re-engineering.

- **Engineering**: use existing verifiers (IFEval) and simple checkers (MCQ exact match; equation validity) only.

**Continual-learning summary metrics (computed from task matrix \(P_{s,i}\)):**
- **AvgAcc**: mean of \(P_{T,i}\) across tasks after the final stage.
- **BWT (Backward Transfer)**: \(\text{BWT}_i = P_{T,i} - P_{i,i}\) (higher is better).
- **FM (Forgetting Measure)**: \(\text{FM}=\frac{1}{T-1}\sum_{i=1}^{T-1}(P_{T,i}-\max_{k\in[i,T]}P_{k,i})\) (closer to 0 is better).

### Experimental Rigor

**Variance & Reproducibility:**
- Main conditions use **seeds = [42, 123, 456]** and report mean±std.

**Validity threats and controls:**
1. **Training-intensity confound (SFT variants)**: RazorSFT uses fallback so every prompt yields exactly one target; match epochs and max target length across Offline SFT and RazorSFT.
2. **Degenerate “mostly-offline” RazorSFT**: track replacement rate \(\rho_t\); treat \(\rho_t<0.5\) as inconclusive.
3. **Verifier bias toward easy prompts**: report replacement rate stratified by base-model correctness on each prompt before training that stage.

**Sanity checks:**
- Verify that best-of-8 inference improves task accuracy without changing FM (since there is no training), validating that forgetting metrics capture training-induced changes.

**Data leakage:**
- Use fixed train/test splits from the Retaining-by-Doing repository and official benchmarks; ensure that prompts used to generate on-policy targets are from the training split only.

**Fair comparison:**
- Keep base model and inference decoding settings fixed across methods for evaluation; for methods requiring sampling during training (GRPO/RazorSFT), log total generated tokens so compute usage can be audited.

## Success Criteria

**Hypothesis (directional):** RazorSFT-8 will (i) outperform Offline SFT on forgetting (FM closer to 0; less negative BWT) and (ii) match GRPO’s retention–adaptation trade-off (FM and AvgAcc) within statistical noise.

**Decision Rule (concrete):**
- **Proceed / accept** if:
  1) \(\text{FM}(\text{RazorSFT})\) is within **1 pooled std** of \(\text{FM}(\text{GRPO})\),
  2) \(\text{AvgAcc}(\text{RazorSFT})\) is within **1 pooled std** of \(\text{AvgAcc}(\text{GRPO})\), and
  3) \(\text{FM}(\text{RazorSFT})\) improves over Offline SFT by **> 1 pooled std**.
- **Refute** if RazorSFT is clearly worse than GRPO: \(\text{FM}(\text{RazorSFT})\) is worse by **>1.5 pooled std** or AvgAcc is lower by **>1 pooled std**.
- **Mechanism outcome**: if OnPolicy-SFT (unfiltered) matches RazorSFT on FM/AvgAcc, conclude that verifier filtering is not the main contributor (the filtering hypothesis is refuted).
- **Inconclusive** if \(\rho_t<0.5\) for any stage in RazorSFT-8.

---

## Impact Statement

If RazorSFT matches GRPO, it provides an SFT-compatible method for continual post-training that can preserve prior capabilities without requiring a full RLVR training stack. This could reduce engineering complexity for teams that repeatedly update LLMs on verifiable tasks (instruction following with rule checkers, multiple-choice QA, programmatically checkable reasoning).
